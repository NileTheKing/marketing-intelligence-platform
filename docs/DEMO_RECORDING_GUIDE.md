# 🎬 Axon Portfolio Demo Recording Guide

이 가이드는 현대오토에버 포트폴리오용 고퀄리티 데모 영상 촬영을 위한 시나리오입니다.
`bash scripts/seed-power-demo.sh`를 실행하면 모든 데이터가 준비됩니다.

---

### 🎥 Scene 1: 전역 대시보드 (플랫폼의 공신력)
- **목표**: 60일간의 안정적인 운영 데이터와 최근 캠페인 성과를 한눈에 보여줍니다.
- **동선**: 
    1. [전역 대시보드](http://localhost:8080/admin/dashboard/overview) 접속.
    2. 우측 상단 기간 필터를 **'60 Days'**로 변경.
    3. 과거부터 쌓여온 매출 그래프와 캠페인 랭킹(Activity 11, 31 등)을 스크롤하며 보여줌.

### 🎥 Scene 2: 코호트 분석 (CRM 엔지니어링 역량)
- **목표**: "데이터를 단순히 쌓는 것을 넘어, 고객 유지율(Retention)을 분석하고 LTV를 관리한다"는 핵심 가치를 전달합니다.
- **동선**:
    1. [Activity 11 코호트 대시보드](http://localhost:8080/admin/dashboard/cohort/11) 접속.
    2. **Retention Heatmap**을 보여주며, 60일 전 유입된 고객들이 주차별로 얼마나 재구매(LTV)하고 있는지 설명.
    3. (멘트 제안): "단순 유입에 그치지 않고, 고객 생애 가치(LTV)를 데이터로 추적하여 마케팅 효율을 고도화합니다."

### 🎥 Scene 3: AI 성과 분석 & 하이브리드 LLM (최신 기술 트렌드)
- **목표**: RAG와 Tool Calling이 결합된 하이브리드 아키텍처의 실무 적용 사례를 보여줍니다.
- **동선**:
    1. [Activity 21 상세 대시보드](http://localhost:8080/admin/dashboard/activity/21) 접속.
    2. 낮은 구매 전환율(5%) 퍼널을 보여줌.
    3. 하단 AI 챗봇에 질문: `"왜 'Spring Maintenance Discount' 활동의 전환율이 낮을까?"`
    4. 로그 터미널을 함께 띄워, LLM이 **API를 스스로 호출(Tool Calling)**하여 데이터를 분석하는 과정을 노출.

### 🎥 Scene 4: 고가용성 실시간 스파이크 (기술적 도전 과제)
- **목표**: 수천 명의 동시 접속자가 몰리는 상황에서도 안정적인 선착순(FCFS) 처리를 증명합니다. 
- **왜 k6인가?**: `seed` 스크립트는 단순히 데이터를 채우는 방식이지만, `k6`는 수십 명의 가상 사용자가 동시에 요청을 쏘는 **부하 상황**을 완벽히 재현합니다. 대시보드 숫자가 '우수수' 떨어지며 스파이킹하는 박진감을 줄 수 있습니다.
- **동선**:
    1. [Activity 11 상세 대시보드](http://localhost:8080/admin/dashboard/activity/11) 접속. (사전에 `seed` 스크립트로 60일치 데이터가 있는 상태)
    2. 상단 **'Real-time'** 위젯(참여자 수, 잔여 수량)에 포커스.
    3. 터미널에서 아래 `k6` 명령을 실행:
       ```bash
       # 50명의 사용자가 30초 동안 맹렬히 응모하는 상황 연출
       k6 run scripts/load-test/k6-fcfs-load-test.js \
         -e ACTIVITY_ID=11 \
         -e VUS=50 \
         -e DURATION=30s \
         -e USE_PRODUCTION_API=false
       ```
    4. 대시보드에서 **남은 수량(Stock)**이 실시간으로 깎이고, 참여자 수가 급증하는 모습을 녹화.

### 💡 촬영 팁
- **화면 구성**: 브라우저와 터미널(로그/k6 실행화면)을 7:3 비율로 분할하여 배치하면 공학적 신뢰도가 높아집니다.
- **포인터**: 마우스 포인터에 하이라이트 효과를 주어 시선의 흐름을 가이드하세요.
- **편집**: 불필요한 로딩 시간은 컷 편집하고, 자막으로 핵심 기술 스택(Kafka, Redis, Tool Calling)을 강조하세요.
