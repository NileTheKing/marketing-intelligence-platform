# Axon JS SDK (Behavior Tracker) 고도화 및 분석 확장 계획

## 1. 개요
현재 Axon JS SDK는 `PAGE_VIEW`와 `CLICK` 기반의 기본 상호작용만 수집하고 있습니다. 본 계획은 유저의 **관심도(Engagement)**와 **경험적 병목(Friction)**을 정교하게 파악하여 마케팅 인사이트를 고도화하는 것을 목적으로 합니다.

---

## 2. 추가 추적 항목 (Key Metrics)

### A. 심층 관심도 측정 (Deep Engagement)
1. **스크롤 깊이 (Scroll Depth)**
   - **설명**: 페이지의 25%, 50%, 75%, 100% 지점을 통과할 때마다 이벤트 전송.
   - **가치**: 유저가 상세 페이지의 핵심 정보(상품 설명, 리뷰 등)를 실제로 읽었는지 판별.
2. **능동적 체류 시간 (Active Dwell Time)**
   - **설명**: 단순히 페이지를 켜둔 시간이 아니라, `Visibility API`를 활용해 탭이 활성화되어 있고 유저가 움직이는 시간만 측정.
   - **가치**: 허수 체류 시간을 제외한 '진짜 관심' 시간 측정.

### B. 사용자 경험 장애 측정 (Friction Analysis)
1. **분노 클릭 (Rage Clicks)**
   - **설명**: 1초 이내에 동일한 요소에 3회 이상 클릭이 발생한 경우.
   - **가치**: 버튼이 작동하지 않거나 로딩이 느려 유저가 답답함을 느끼는 지점(병목) 발견.
2. **데드 클릭 (Dead Clicks)**
   - **설명**: 인터랙티브 요소(버튼, 링크)가 아닌 곳을 클릭하는 행위.
   - **가치**: UI 설계 미흡으로 유저가 오인하는 구역 확인.

### C. 마케팅 전환 의도 측정 (Conversion Intent)
1. **양식 중단 (Form Abandonment)**
   - **설명**: 회원가입이나 주문 폼에서 특정 필드까지 입력하고 이탈한 경우.
   - **가치**: 어느 단계에서 유저가 심리적 거부감을 느끼는지 파악 (예: 전화번호 요구 단계 등).
2. **장바구니 잔류 (Cart Persistence)**
   - **설명**: 장바구니에 담은 후 구매하지 않고 이탈하는 패턴 수집.

---

## 3. 기술적 구현 전략 (Engineering Excellence)

### 1) 데이터 전송 신뢰성 확보 (`navigator.sendBeacon`)
- **문제**: 유저가 페이지를 닫는 순간 발생하는 이벤트(체류 시간 등)는 일반 `fetch`로 보내면 유실될 확률이 높음.
- **해결**: 페이지 종료 시점에 비동기적으로 데이터를 전송하는 `sendBeacon` API를 사용하여 유실률 0%에 도전.

### 2) 클라이언트 성능 최적화 (Throttling & Debouncing)
- **문제**: 스크롤 이벤트는 밀리초 단위로 수천 번 발생하여 브라우저 부하 유발.
- **해결**: `Throttling`(100ms 단위 실행)을 적용하여 메인 스레드 점유율 최소화.

### 3) 익명-식별자 통합 (Identity Stitching)
- **설명**: 비로그인 상태의 `sessionId`와 로그인 후의 `userId`를 Kafka 레이어에서 매핑하여, 유저의 과거 익명 활동 이력을 현재 계정에 결합.

---

## 4. 분석 및 활용 시나리오 (AI 연동)
- **AI 보고서 연동**: Gemini 에이전트가 "평균 스크롤 깊이가 50% 미만인 캠페인은 혜택 설명을 상단으로 배치하세요"와 같은 실행 가능한 액션 제안.
- **LTV 예측**: 체류 시간과 스크롤 깊이가 깊은 유저군을 '고관여 유저'로 분류하여 타겟팅 우선순위 할당.

---

## 5. 단계별 로드맵
- **Phase 1 (즉시)**: 스크롤 깊이 및 체류 시간 기본 로직 추가.
- **Phase 2 (고도화)**: `Rage Click` 감지 및 `sendBeacon` 적용.
- **Phase 3 (분석)**: 수집된 지표를 대시보드 시계열 그래프로 시각화.
